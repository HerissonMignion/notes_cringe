

En winform, il y a la class abstraite Control et ControlContainer. Tout les contrôles graphique extends l'une de ces deux classes parce que le code est mieux organisé comme cela. La raison pour laquelle nous prenons la décision de faire les classes abstraites Control et ControlContainer est que le code est plus simple à travailler après, plus beau. Nous imaginons dans notre tête les contrôles graphique que l'on désire programmer avec les relations entre eux (https://en.wikipedia.org/wiki/Entity%E2%80%93relationship_model), nous examinons tout les points qu'ils ont en commun, parce que nous avons compris qu'il ne faut pas écrire inutilement les mêmes lignes de code plusieurs fois (ctrl-c, ctrl-v), nous faisons une/deux classes abstraites qui contiendra les choses qui sont partagés entre tout les contrôles graphiques.
